以下是针对《198. 打家劫舍》这道题的详细拆解：

### （1）用白话文解释题目意思及生僻概念

**题目白话文解释：**
有一排挨在一起的房子，每个房子里都放着一定数量的现金。现在的目标是拿走尽量多的现金。但是有一个硬性规则限制：**绝对不能同时拿走相邻两家房子的现金**，否则就会触发警报。问题要求算出来，在不触发警报的前提下，一趟下来最多能拿到多少钱。

**生僻概念解释：**

  * **非负整数数组**：一个列表里的数字全都是大于或等于 0 的整数。在这里代表每个房子里存放的具体金额数。
  * **相邻的房屋**：指列表中紧挨着的两个数字（例如第 1 个和第 2 个，第 2 个和第 3 个）。

-----

### （2）解题思路是怎么来的（基于代码反推）

这段代码采用的是\*\*动态规划（状态压缩）\*\*的思想。

1.  **核心抉择（偷还是不偷）**：
    走到任意一个房子面前，都面临两个选择：

      * **选择一：偷这家。** 既然偷了这家，那么紧挨着它的前一家肯定不能偷。所以，此时能拿到的最大总金额 = **“到上上家为止的最大收益” + “这家的金额”**。
      * **选择二：不偷这家。** 既然不偷这家，那么总金额就不会增加，此时能拿到的最大总金额 = **“到上一家为止的最大收益”**。
        为了拿到最多的钱，只需要把这两个选择的结果比个大小，挑那个数字更大的方案。

2.  **滚动记录（精简记忆）**：
    从头走到尾的过程中，要想决定当前房子的最佳方案，其实只需要知道前面两个数据：一个是“到上一家为止的最大收益”，另一个是“到上上家为止的最大收益”。更早之前的数据已经没用了。
    因此，代码中设立了两个变量 `cur`（代表上一家的最大收益）和 `pre`（代表上上家的最大收益）。每往前走一个房子，就把这两个变量像车轮一样往前滚动更新一次，直到走完全部房子。

### （3）代码框内的详细注释

```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        cur, pre = 0, 0                      # 初始化两个滚动变量：cur代表到前一个房屋的最大收益，pre代表到前两个房屋的最大收益
        for num in nums:                     # 从左到右依次遍历每个房屋里的金额
            cur, pre = max(pre + num, cur), cur  # 计算走到当前房屋时的最大收益更新给cur，同时把旧的cur退位赋值给pre
        return cur                           # 遍历完所有房屋后，cur里存的就是最终能拿到的最高金额
```

改成下面的更直观看出前一个previous，前两个pre_previous和当前current：
```python
class Solution:
    def rob(self, nums: List[int]) -> int:
        previous, pre_previous = 0, 0
        for current in nums:
            previous, pre_previous = max(current + pre_previous, previous), previous
        return previous
```

### （4）每一行代码的详细中文解释

  * `class Solution:`
      * 定义一个类，用于封装解题的算法方法。
  * `def rob(self, nums: List[int]) -> int:`
      * 定义一个名为 `rob` 的函数。接收一个整数列表 `nums`（代表每个房屋的金额），并明确该函数最后会返回一个代表最高金额的整数。
  * `cur, pre = 0, 0`
      * 初始化两个用来记录状态的滚动变量。`cur`（current 的缩写）用于记录如果只考虑到“上一个房屋”时能获取的最高金额；`pre`（previous 的缩写）用于记录如果只考虑到“上上个房屋”时能获取的最高金额。由于还没开始拿任何东西，初始状态均为 0。
  * `for num in nums:`
      * 开启一个循环，按顺序依次取出列表中每一个房屋存放的金额，将其赋值给临时变量 `num`。
  * `cur, pre = max(pre + num, cur), cur`
      * 这是整段代码最核心的状态转移操作。等号右边进行计算：如果拿当前房屋，收益是 `pre + num`；如果不拿当前房屋，收益是目前的 `cur`。用 `max()` 函数选出两者中较大的一个，这个最大值就是截止到当前房屋的最新最高收益。
      * Python 的逗号赋值是同时发生的。算出的最新最高收益会被赋值给等号左边的新 `cur`；而等号右边那个旧的 `cur` 值，会被原封不动地传递给 `pre`。这样就完成了变量向前滚动的准备工作。
  * `return cur`
      * 当所有房屋都遍历完毕，循环结束。此时，经过一路滚动比对筛选，`cur` 中保存的正是整条街能拿到的最高总金额。将其作为最终结果返回。

# 疑问与解答：在这个代码里面，感觉拿的就是两个相邻的屋子，直接触发警报了？

这个直觉非常普遍！很多初学者在看这段代码时都有同样的疑惑。

因为代码里有一个 `for num in nums:` 循环，一直在挨个、连续地读取每一个房子的金额。从视觉上看，就像是一路“平推”过去，确实很难直观地感觉到“跳过了相邻的房子”。

这个“不能偷相邻房屋”的硬性规定，其实深深地隐藏在 **`pre + num`** 这一小段加法里。

下面把代码里的这层“障眼法”彻底剥开：

### 1\. 变量的真实身份

要看懂怎么“跳”的，必须先明确 `pre` 和 `cur` 到底记录的是什么状态。
假设现在循环走到了第 3 个房子（金额是 3），准备做决定：

  * **`num`**：当前第 3 个房子里的钱。
  * **`cur`**：这是走到**第 2 个房子**为止，能拿到的最高金额。如果这个最高金额里**包含了第 2 个房子的钱**，那现在就**绝对不能**再加第 3 个房子的钱了。
  * **`pre`**：这是走到**第 1 个房子**为止，能拿到的最高金额。既然是截止到第 1 个房子，那就意味着**绝对没有碰过第 2 个房子**。

### 2\. 核心代码的潜台词

代码 `max(pre + num, cur)` 的本质，是在做两个“平行宇宙”的对决：

  * **平行宇宙 A（偷当前房子）** ➔ 对应代码 **`pre + num`**
    既然决定要偷当前的房子（第 3 个），就必须和前一个房子（第 2 个）隔开。
    怎么隔开？办法就是**强行使用上上个房子的最高记录 `pre`**。
    所以 `pre + num` 的物理意义是：继承第 1 个房子的收益，**跨过（不偷）第 2 个房子**，把第 3 个房子的钱加上。**跳跃就发生在这里！**

  * **平行宇宙 B（不偷当前房子）** ➔ 对应代码 **`cur`**
    如果发现上一个房子（第 2 个）实在太有钱了，为了保住这笔巨款，只能忍痛放弃当前的房子（第 3 个）。
    此时不加 `num`，直接把前一个房子的最高记录 `cur` 延续下来。

### 3\. 用 `[1, 2, 3, 1]` 慢镜头回放“跳跃”瞬间

注意看第 3 步，跳跃是如何发生的：

| 当前走到 | 变量状态 | `max(pre + num, cur)` 的抉择过程 | 谁赢了？ |
| :--- | :--- | :--- | :--- |
| **第 1 间**<br>金额: 1 | `pre = 0`<br>`cur = 0` | 偷：`0 + 1 = 1`<br>不偷：`0` | **偷了第 1 间**<br>更新后：`cur=1, pre=0` |
| **第 2 间**<br>金额: 2 | `pre = 0`<br>`cur = 1` | 偷：使用上上家(0) + 当前(2) = `2`<br>不偷：保留上一家(1) = `1` | **偷了第 2 间** (相当于放弃第1间)<br>更新后：`cur=2, pre=1` |
| **第 3 间**<br>金额: 3 | `pre = 1`<br>`cur = 2` | **偷：** 使用上上家记录(`pre`是1，来自第1间) + 当前(3) = `4`。<br>*(看！这里直接拿第1间的钱加上了第3间的钱，完美避开了第2间！)*<br>**不偷：** 保留上一家最高记录(`cur`是2，来自第2间) = `2`。 | **偷了第 1、3 间**<br>更新后：`cur=4, pre=2` |
| **第 4 间**<br>金额: 1 | `pre = 2`<br>`cur = 4` | 偷：使用上上家(2) + 当前(1) = `3`<br>不偷：保留上一家最高记录(4，来自第1+3间) = `4` | **不偷第 4 间** (保住1、3间的钱)<br>更新后：`cur=4, pre=4` |

-----

通过这种“错位相加”的方式，代码在不知不觉中就强制执行了“不能相邻”的规则。
